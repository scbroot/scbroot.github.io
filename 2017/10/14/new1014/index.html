Raspberry, Raspberry pi, 라즈베리, 라즈베리 파이

라즈베리 파이(Raspberry pi)

노트북에 우붕투 서버를 설치 하여 사용하고 있었다
추석이전에 뭘 잘 못 사용하여 몇가지 서비스 되지 않아 서버를 다시 설치 하였다.

작동중인 서버안에는 git 서버도 같이 돌고 있었다
그래서 이번에 설치 하면서 git서버를 분리할 필요성을 느끼며
라즈베리 파이를 구매하였다

현재로서는 git만 활용할 예정이다
우선적으로 결제와 설치에 중점을 두고 검색을 했봤다

윈도우 버젼의 설치 방법은 많았는데
맥에서 설치 하는 밥법에 대한 자료는 많지 안았다

판매 사이트에서 제공하는 메뉴얼 대로 일단 해봤다
메뉴얼 대로 작업했으나
etcher(https://etcher.io/)에서 제공하는 메모리카드 굽는 프로그램에서
메모리카드 하나를 날렸다
논리적 오류를 인해 더 이상 메모리카드를 인식 하지 못하게 되었다.
이런 엿같은 경우가..

구글에서 검색하 맥에서 설치 하는 방벙을 검색해보니
맥 적용 인스톨러(https://github.com/RayViljoen/Raspberry-PI-SD-Installer-OS-X)가 있어서 그걸로 설치하려 했으나 메모리카드를 구매하기 늦은 시간이라
집에 있던 고프로에 있던 메모리 카드로 다시 설치 했다.
결국 고프로 메모리카드를 하나 주문했다.

설치가 잘 되었지만 가뷴이 그렇게 깔끔하지 못하다.

lite 버젼으로 재 설치 할까 고민 중이다.
git서버로만 사용한다면 굳이 GUI 형태의 os가 불필요 할것 같다.

프로토콜은 공유기 포트포워딩으로 해결 가능하니 굳이 라즈베리에서 세팅을 할 필요는 없을것 같다.
